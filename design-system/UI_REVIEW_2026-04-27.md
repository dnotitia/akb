## P0 ÔÇö CRITICAL (fix before next ship)

### 1. Destructive actions use `window.confirm()` instead of an accessible dialog
**Pattern violation of MASTER ┬º7.**
- `pages/settings.tsx:112` (delete user + vaults)
- `pages/settings.tsx:406` (revoke PAT)
- `pages/home.tsx:518` (revoke PAT, inline)
- `pages/publications.tsx:90` (unpublish)

**Why critical**: native `confirm()` traps focus weirdly, isn't styled, can't be dismissed with mouse-outside on some browsers, isn't screen-reader-friendly, and breaks the editorial chrome.
**Fix**: build a single `<ConfirmDialog>` from existing `ui/dialog.tsx`, accept `{ title, body, confirmLabel, variant: "destructive" }`. Replace all 4 call sites.

### 2. Touch targets below 44px on mobile-tappable controls
- `ui/button.tsx:33` ÔÇö `sm: h-8` (32px), `default: h-9` (36px), `icon: h-9 w-9` (36px)
- `ui/input.tsx:10` ÔÇö `h-10` (40px)
- `ui/dialog.tsx` close button ÔÇö 32px (audit-reported)
- `components/vault-shell.tsx:138` ÔÇö tree toggle `h-6 w-6` (24px)
- `components/vault-explorer.tsx:194` ÔÇö filter input `h-7` (28px)
- `components/vault-nav.tsx:67` ÔÇö filter input `h-6` (24px)
- `pages/graph.tsx:734` ÔÇö layout switcher `px-2.5 py-1.5` (Ôëê30px)

**Why critical**: violates Apple HIG (44pt) and Material (48dp) on phones; the layout switcher and tree toggle in particular are dense touch zones.
**Fix**: keep visual height for desktop density, but add `min-h-[44px]` *or* `hitSlop` padding (`-m-2 p-2`) on mobile breakpoints. Lift `Button.icon` to `h-10 w-10` and add `sm:h-9 sm:w-9` for desktop downsize.

### 3. Long lists are not virtualized
- `components/vault-explorer.tsx` tree rows (~line 234)
- `pages/vault.tsx` recent activity & commit log
- `pages/graph.tsx` selected-node / candidates list
- `pages/document.tsx` history tab

**Why critical**: a vault with 5k notes locks Chrome on first paint. Already happens on internal large vaults.
**Fix**: introduce `@tanstack/react-virtual` for the tree and history tabs. Keep recent (capped at 8) as-is.

### 4. Markdown / table content has no horizontal-scroll guard on mobile
- `pages/document.tsx:206` ÔÇö prose container has `style={{ maxWidth: "100%" }}` but inner `<pre>` and wide `<table>` will overflow
- `pages/table.tsx:134-171` ÔÇö wide tables overflow without a labeled scroll container
- `pages/settings.tsx:556` ÔÇö snippet `<pre overflow-x-auto>` causes long horizontal scroll on mcp.json

**Why critical**: 375px viewport breaks; horizontal scroll fights vertical reading.
**Fix**: wrap each `<pre>` in `overflow-x-auto` + `whitespace-pre-wrap break-all` (already correct on home connect snippet ÔÇö port the pattern). For tables: add `<div role="region" aria-label="ÔÇª" tabIndex={0}>` wrapper and a fade-edge indicator when scrollable.

### 5. Graph (cytoscape) reads CSS vars without fallback + no keyboard a11y
- `pages/graph.tsx:79-94` ÔÇö colors come from `getComputedStyle(--color-*)`; if a var is missing the graph renders transparent nodes
- `pages/graph.tsx:803` ÔÇö canvas has `role="application"` but no `aria-label`, no keyboard navigation, no node-level focus

**Why critical**: graph is one of AKB's signature features and is currently unusable for keyboard / screen-reader users; also susceptible to silent failure if a token is renamed.
**Fix**: (a) add `|| "#0a0908"` fallback in the var-read; (b) add a sibling text representation (a focusable list of nodes with same data) in DOM for screen readers; (c) `aria-label="Document relation graph. Use Tab to focus nodes, Enter to open."`

## P1 ÔÇö HIGH (next sprint)

### 6. Loading states are bare text ÔÇö cause layout shift & feel hung
- `pages/document.tsx:108` ÔÇö "LoadingÔÇª" text without skeleton
- `pages/vault.tsx:227` ÔÇö bare "LOADINGÔÇª" string
- `components/vault-explorer.tsx:206` ÔÇö "ÔÇö LOADING ÔÇö" text
- `pages/search.tsx:296` ÔÇö skeleton + "ÔÇö RerankingÔÇª" but doesn't say cached vs fresh

**Why**: empty space then sudden content = high CLS; silent reranks confuse.
**Fix**: extend `ui/skeleton.tsx` with row/list/prose presets. Reserve column widths matching final layout. Add an `aria-live="polite"` "Reranking resultsÔÇª" announcement.

### 7. Form labels: placeholder-only and uppercase mangling
- `pages/vault-new.tsx:82` ÔÇö hint rendered as `coord` (all caps) for help text ÔåÆ unreadable
- `pages/vault-new.tsx:123` ÔÇö `error.toUpperCase()` shouted at users
- `pages/file.tsx:77-85` ÔÇö `error.toUpperCase()` in role="alert" ÔåÆ screen-readers shout
- `pages/search.tsx:199` ÔÇö visible label missing, only `aria-label="Query"`
- `pages/auth.tsx:523-537` ÔÇö Field label/htmlFor coupling weak

**Fix**: (a) introduce a `<HelpText>` with sentence-case `text-xs text-foreground-muted`; (b) never wrap user-facing strings in `.toUpperCase()`; (c) add a visible `<Label>` to the global header search (can be `sr-only` if design rejects it visually, but the search input on `/search` should have a visible "Query" label).

### 8. Dialog & destructive copy don't explain consequences
- `pages/home.tsx:519` ÔÇö confirm string only mentions "loses access" but not which agents
- `pages/settings.tsx:113` ÔÇö delete user copy needs irreversibility + data-loss warning
- `pages/document.tsx:230-252` ÔÇö Publish Ôåö Unpublish toggle button has no visual differentiation

**Fix**: standardize dialog copy template:
```
Title: Revoke "{name}"?
Body:  Any agent currently using this token will lose access immediately.
       This cannot be undone.
Confirm: "Revoke" (variant="destructive")
```
For Publish toggle: change button to `outline` when published with `accent` "Update" + ghost "Unpublish" sibling.

### 9. State persistence missing across tabs / navigation
- `pages/document.tsx:271` ÔÇö Tabs `defaultValue="outline"` resets on every visit
- `components/vault-shell.tsx` ÔÇö sidebar collapsed state not persisted
- `pages/vault.tsx` ÔÇö recent commit log scroll position not preserved when leaving doc and returning

**Fix**: persist tab + sidebar in `localStorage` keyed by `vault:name`. React Router's `useLocation` state can hold scroll for the doc-back navigation.

### 10. Theme toggle & PAT mint UX gaps
- `components/theme-toggle.tsx:24` ÔÇö 36px target (P0 above also)
- `pages/settings.tsx:623` ÔÇö theme value not announced via `aria-live`
- `pages/home.tsx:430-454` ÔÇö fresh PAT box: show/hide + copy work, but no auto-dismiss timer & no "Save before leaving" warning

**Fix**: add `aria-live="polite"` to theme value indicator. For fresh PAT, show a 30s countdown + warn on `beforeunload` if active token hasn't been copied.

## Cross-cutting recommendations

### A. Mobile sweep
The product is desktop-first by design (it's a console for agents), but
the public publication route (`/p/:slug`) and the auth/settings pages
**will** be hit on phones. Run a 375px audit on:
- `/p/:slug` (public publication)
- `/auth`
- `/settings`
- `/` (home)

ÔÇªand ensure the right-rail collapses *under* the main column on `lg:`,
not at `xl:`. Currently `home.tsx:213` uses `lg:grid-cols-[1fr_320px]`,
which is correct ÔÇö verify the same pattern across `vault.tsx`,
`document.tsx`, `graph.tsx`.

### B. Build a shared primitive layer
Recurring patterns that should be extracted into `components/ui/`:
- `<SectionHeader ┬ºnumber title count>` ÔÇö used 8+ times
- `<ConfirmDialog>` ÔÇö 4 call sites of `confirm()`
- `<HelpText>` & `<ErrorText>` ÔÇö currently hand-rolled per page
- `<RailCard title={coord-ink}>` ÔÇö used in home, settings
- `<ListRow leadCoord title path tail>` ÔÇö home, vault, publications all reimplement this

This will (a) enforce design rules at the type level, (b) shrink each
page by ~30%, (c) catch the touch-target / a11y regressions in one
place.

### C. Dark mode sanity pass
Pages I'd visually QA in dark mode specifically:
- `/vault/:name/graph` ÔÇö cytoscape colors (P0 #5)
- `/p/:slug` ÔÇö prose dark variant
- `/settings` admin user table (line 641+) ÔÇö many cells, contrast risk
- `/document` history tab ÔÇö commit hash + author chips contrast

### D. Accessibility automation
Add `axe-core` to vitest setup (`test-setup.ts`) and run a smoke test
on each page's mounted output. Catches missing labels, contrast, ARIA
on every PR.
